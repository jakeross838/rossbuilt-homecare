# Plan 02-01: Project Setup & Core Infrastructure

## Overview

Initialize the Vite + React admin app with monorepo structure, Tailwind CSS, and core dependencies.

## Prerequisites

- Phase 1 complete (database schema exists)
- Node.js installed

## Tasks

### 1. Initialize Monorepo Structure

Create root workspace with apps and packages directories:

```
home-care-os/
├── apps/
│   ├── admin/         # This phase
│   ├── client-portal/ # Future
│   └── inspector/     # Future
├── packages/
│   ├── shared/        # Future
│   ├── database/      # Future
│   └── api/           # Future
├── supabase/          # Already exists
└── .planning/         # Already exists
```

### 2. Create Admin App with Vite

```bash
cd apps/admin
npm create vite@latest . -- --template react-ts
```

### 3. Install Dependencies

Core dependencies:
- `@supabase/supabase-js` - Supabase client
- `zustand` - State management
- `@tanstack/react-query` - Data fetching
- `@tanstack/react-table` - Data tables
- `react-router-dom` - Routing
- `react-hook-form` - Forms
- `@hookform/resolvers` - Form validation
- `zod` - Schema validation
- `date-fns` - Date utilities
- `lucide-react` - Icons

UI dependencies:
- `tailwindcss`, `postcss`, `autoprefixer`
- `class-variance-authority`, `clsx`, `tailwind-merge`
- Radix UI primitives (dialog, dropdown, select, tabs, toast, tooltip, popover, avatar, separator, checkbox)

### 4. Configure Tailwind CSS

Setup with shadcn/ui compatible config including:
- CSS custom properties for theming
- Ross Built brand colors (rb-green, rb-sand)
- Border radius variables
- Dark mode support

### 5. Create Core Utility Files

- `lib/utils.ts` - cn() helper for class merging
- `lib/supabase.ts` - Supabase client initialization
- Environment variables setup (.env.local)

### 6. Copy TypeScript Types

Copy `database.types.ts` from supabase folder to admin app types folder.
