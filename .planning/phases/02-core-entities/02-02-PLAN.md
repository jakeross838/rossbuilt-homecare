# Plan 02-02: shadcn/ui Components

## Overview

Create the base UI component library using shadcn/ui patterns with Radix UI primitives.

## Prerequisites

- Plan 02-01 complete
- Tailwind CSS configured

## Tasks

### 1. Create UI Component Directory

```
src/components/ui/
```

### 2. Implement Base Components

Each component follows shadcn/ui patterns with:
- TypeScript interfaces
- Tailwind styling with CVA variants
- Radix UI primitives where applicable
- Proper accessibility

**Components to create:**

1. **button.tsx** - Primary, secondary, destructive, outline, ghost variants
2. **input.tsx** - Text input with error state support
3. **label.tsx** - Form label
4. **textarea.tsx** - Multi-line input
5. **select.tsx** - Dropdown select using Radix
6. **checkbox.tsx** - Checkbox using Radix
7. **dialog.tsx** - Modal dialog using Radix
8. **dropdown-menu.tsx** - Context menu using Radix
9. **table.tsx** - Data table components
10. **badge.tsx** - Status badges
11. **card.tsx** - Card container with header, content, footer
12. **toast.tsx** - Toast notifications using Radix
13. **tabs.tsx** - Tab navigation using Radix
14. **separator.tsx** - Visual separator
15. **skeleton.tsx** - Loading skeleton
16. **avatar.tsx** - User avatar using Radix
17. **popover.tsx** - Popover using Radix
18. **tooltip.tsx** - Tooltip using Radix

### 3. Create Shared Components

```
src/components/shared/
```

1. **loading-spinner.tsx** - Animated loading indicator
2. **empty-state.tsx** - Empty state with icon, title, description, action
3. **confirm-dialog.tsx** - Confirmation modal with destructive variant
4. **search-input.tsx** - Search input with icon
5. **data-table.tsx** - Reusable data table wrapper
